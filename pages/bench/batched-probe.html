Batched probe

Swing 2 spike. Coalesces N probe requests into a single POST /preimage/probe against the dev server, one RTT regardless of batch size. Compares directly against strategy: 'auto' (which resolves to range on this origin) over the same N-URL workload. Reports wall time, bytes on wire, per-probe distribution, and batch count.

This is the phase-1 direct client — no service worker yet. The consumer opts in via configureBatchedProbe({ endpoint }) + strategy: 'batched'. Phase 3 would intercept normal image probes transparently via an SW; phase 4 binds the same wire to WebTransport for zero-RTT session reuse.
