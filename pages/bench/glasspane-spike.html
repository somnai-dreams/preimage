glasspane spike

Phase 0 feasibility gate for the glasspane swing. Runs the same tile workload through two render backends — DOM (createVirtualTilePool + <img> elements, the current approach) and GPU (createGlasspaneScene + instanced WebGPU draw) — and reports memory, sustained framerate, scroll jank, and time-to-first-tile. The doc's gate criteria are: GPU memory < 50% of DOM on 10k tiles, sustained ≥ 55fps on 2000 tiles, time-to-first-tile within 100ms of DOM.
