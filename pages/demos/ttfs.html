What this shows

Both APIs return the same {width, height}. The delta is time-to-return. Here the image lives in-memory as a Blob, so the transfer term is zero; the delta is pure decode vs byte-parse.

For remote images the difference is much larger: prepare() waits for the full transfer AND the decode; prepareFast() returns after the first ~2KB arrive.

Headline numbers

5MB JPEG over 10 Mbps:
  prepare():       ~4000ms
  prepareFast():   ~150ms

4000×3000 PNG Blob:
  prepare():       ~80ms  (decode only)
  prepareFast():   ~5ms   (probe IHDR)

When prepareFast falls back

AVIF, HEIC, or truncated bytes trigger the createImageBitmap fallback path — still faster than a round trip because the bytes are already in memory, but not the "first-2KB" win.
