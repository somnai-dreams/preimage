Time-to-first-sizing — what does measuring earlier actually look like?

Three panels load the same image at the same instant. Each uses a different strategy for knowing the image's dimensions. The blue dashed frame appears the moment each panel knows the image's aspect ratio — watch how much later it appears on the left panel. The image fades in when the browser fully decodes it.

Naive <img src> reflow

No dimensions declared. Layout reserves 0 space until the image fully loads.
This paragraph sits below the image. With no declared dimensions, the naive <img> starts at zero height and this text jumps down when the real size arrives.

Native <img width height> stable

Author declared the aspect ratio. Browser reserves the frame from t=0 — but you had to know the dimensions ahead of time.
This paragraph sits below the image. The browser uses the width and height HTML attributes to reserve the aspect-ratio box at first paint.

await prepare(src) stable

Preimage streams the first bytes and probes the header. Dimensions arrive fast; you never had to know them in advance.
This paragraph sits below the image. The measured frame appears as soon as prepare() resolves, long before the full image has been transferred or decoded.
