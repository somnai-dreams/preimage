Time to first sizing — one photo, three strategies

The same fresh photo loaded from picsum three ways in parallel, cache-busted so the browser makes a real network request each run. Each panel reports when it knows the image's dimensions and when the image finishes painting. The gap between those two events is the library's visible window.

Naive <img src>

reflows

dimensions unknown until the image is fully decoded

Paragraph below the image. With nothing declared, the browser reserves zero space and this text jumps when the image's natural size arrives.

Native <img width height>

stable

dimensions declared as HTML attributes — only possible if the author knew them

Paragraph below the image. The width/height attributes let the browser reserve an aspect-ratio box from first paint.

Preimage prepare(src)

stable

dimensions probed from the first bytes — no author declaration required

Paragraph below the image. Preimage streams the fetch, parses the header as bytes arrive, and reserves the frame long before the full transfer completes.
