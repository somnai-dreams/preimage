Pretext rich-inline — inline images in a chat bubble

Three inline images interleaved with flowing text. Pretext's rich-inline walker treats each image as an atomic pill and wraps around it naturally. Resize the bubble — lines re-break and images shuffle to the right slots.

What this shows

Pretext's RichInlineItem has text, font, break, and extraWidth. The extraWidth field is caller-owned; it's how pretext lets pill chrome contribute to line width without needing a font.

inlineImage() returns a RichInlineItem whose text is a single word joiner (U+2060 — invisible, zero-width, survives pretext's trim) and whose extraWidth is the measured image's rendered width plus any caller-supplied chrome. Pretext packs it like any other atomic item.

Without preimage

Pretext's rich-inline is text-only. To mix in images you'd fork rich-inline, or stop using pretext inside the bubble.

With preimage

const items = await resolveMixedInlineItems([
  { text: 'Pushed a fix to ', font: FONT },
  { kind: 'image', src: iconSrc,
    options: { font: FONT, height: 20,
               extraWidth: 6 } },
  { text: ' preimage.', font: FONT },
])
const prepared = prepareRichInline(items)
