Pretext rich-inline — inline images in a chat bubble

Three tiny images interleaved with flowing text. Pretext's rich-inline walker treats each image as an atomic pill and wraps around it naturally. Resize the bubble — lines re-break and images shuffle to the right slots.

What this shows

Pretext's RichInlineItem has a text, font, break, and extraWidth. The extraWidth field is caller-owned — it's how pretext lets pill chrome contribute to line width without needing a font.

Preimage's inlineImage() returns a RichInlineItem whose text is a zero-width space (survives pretext's whitespace trim, measures 0px wide) and whose extraWidth is the measured image's rendered width. Pretext packs it like any other atomic item.

Without preimage

Pretext's rich-inline is text-only. To mix in images you'd have to fork rich-inline, or stop using pretext inside the bubble. Either way you lose pretext's line-break accuracy for the text portions.

With preimage

const items = await resolveMixedInlineItems([
  { text: 'Pushed a fix to ', font: FONT },
  { kind: 'image', src: iconSrc,
    options: { font: FONT, height: 20,
               extraWidth: 6 } },
  { text: ' preimage.', font: FONT },
])
const prepared = prepareRichInline(items)

Iterate fragments with pretext's walker. Fragments whose source item is an InlineImageItem render as images, everything else as text.
