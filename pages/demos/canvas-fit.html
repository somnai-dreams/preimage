Canvas fit — CSS object-fit for pixels CSS can't reach

Drawing an image to a canvas is the clearest case where CSS does not help you. drawImage(img, 0, 0, w, h) stretches. object-fit is a CSS property; canvas does not honor it. You have to compute the fitted rect yourself.

naive drawImage(img, 0, 0, 480, 270)
—
preimage fitRect
—

What this shows

Left canvas: what happens when you forget fit math — a 1600×900 image crammed into a 480×270 box. The image stretches and squeezes. Right canvas: the same image, drawn at the rect fitRect() returns for the chosen fit mode. Centered, preserved aspect ratio, same numbers the browser would apply to a DOM <img>.

Without preimage

You hand-roll the fit math every time. Every canvas-based renderer (chart libraries, game engines, PDF/SVG exporters, custom avatar generators) re-derives the same min(boxW/natW, boxH/natH) ladder. Often incorrectly; often without the measurement cache.

With preimage

const p = await prepare(src)
const r = layout(p, 480, 270, 'cover')
ctx.drawImage(img, r.offsetX, r.offsetY, r.width, r.height)

One decode, cached aspect ratio, arithmetic on every resize. All five CSS fit modes supported.
