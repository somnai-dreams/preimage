Canvas fit — object-fit for pixels CSS can't reach

Drawing to a canvas is the clearest case where CSS does not help. drawImage(img, 0, 0, w, h) stretches. object-fit is a CSS property; canvas does not honor it. You have to compute the fitted rect yourself.

naive — drawImage(img, 0, 0, 480, 270)
—
preimage — layout(prepared, 480, 270, fit)
—

What this shows

Left canvas: 3:4 portrait crammed into a 480:270 box with no fit math. Right canvas: fitRect()'s returned rect for the chosen mode, centered with preserved aspect ratio — the same numbers the browser would apply to a DOM <img>.

Without preimage

Every canvas-based renderer reinvents the min(boxW/natW, boxH/natH) ladder. Often incorrectly, often without a measurement cache.

With preimage

const p = await prepare(src)
const r = layout(p, 480, 270, 'cover')
ctx.drawImage(img, r.offsetX, r.offsetY, r.width, r.height)

One decode, cached aspect ratio, arithmetic on every resize. All five CSS fit modes supported.
