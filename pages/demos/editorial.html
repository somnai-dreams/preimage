Editorial — same layout engine, different measurement

Same article, same three floated figures, same flowColumnWithFloats engine on both sides. The only difference is where the figure dimensions come from. Left: native <img> with dims read after onload — pretext has to re-run its flow each time a figure loads, so text shifts under existing lines. Right: prepare() returns dims from the first ~2KB of each image, so pretext flows once with correct rects and nothing moves.
