Container — .prei wrappers, 128-byte deterministic probe

Each image on this page is served through the .prei synthesis endpoint: the dev server reads the underlying PNG/JPEG, runs the existing header parser once, and prepends a 128-byte preimage container carrying width, height, hasAlpha, isProgressive, format, byteLength, and a CRC32. The client asks for Range: bytes=0-127 via strategy: 'container', parses the prefix deterministically, and has everything needed to lay out without touching a format-specific byte parser at all.

Compare against strategy: 'range' (4KB fetched, format parser walked) and strategy: 'img' (full <img> + poll). Byte counts and timings are reported per strategy. All three race against fresh cache-bust URLs so the browser HTTP cache can't collapse runs.
