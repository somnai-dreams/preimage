Sidecar — Preimage-* headers and .prei files

One wire format in two physical shapes. A preimage-aware origin sets Preimage-Width, Preimage-Height, and optional flags as HTTP response headers on every image. Client does a HEAD and parses the headers — zero body bytes, one RTT, no format sniffing. A static origin that can't set custom headers drops a photo.jpg.prei text file next to each image, same Key: Value layout; client fetches the sidecar and parses it identically. The image file is never modified and never wrapped.

This page runs both paths plus the baseline range probe against the same photo set. All three assume the dev server is acting as a cooperating origin — it attaches Preimage-* headers to every image response and synthesises .prei files from the real underlying assets on the fly.
