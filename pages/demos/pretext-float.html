Pretext float — text wraps around a measured image

Drag the sliders. Each move re-flows the paragraph and re-positions the figure. The only things running are pretext's line-break arithmetic and preimage's fitRect. No DOM measurement, no getBoundingClientRect, no decode in the loop.

—

What this shows

Pretext's layoutNextLineRange(prepared, cursor, maxWidth) cursor loop exists specifically so a caller can narrow the available width line by line. Its README shows an example with a floated image — but stops at "you bring the image's width and height from somewhere."

Preimage's flowColumnWithFloats is the "somewhere." It solves each float's rect from the current column width, then drives pretext's cursor with the correct maxWidth on every line.

Without preimage

You load the <img>, wait for onload, read naturalWidth/naturalHeight, decide how to fit it to the column, then write the cursor loop yourself. On canvas/SVG/WebGL render targets, CSS float isn't available at all — the driver is the only path.

With preimage

const image = await prepare(src)
const text = prepareWithSegments(article, FONT)

const { items, totalHeight } =
  flowColumnWithFloats({
    text,
    columnWidth,
    lineHeight: 26,
    floats: [{ image, side: 'right',
               top: 26, maxWidth: 280,
               maxHeight: 220, gapX: 16 }],
  })

items is a flat stream of { kind: 'line' } and { kind: 'float' } placements with absolute (x, y, width, height). Render to DOM, canvas, SVG, or WebGL — the math is the same.
