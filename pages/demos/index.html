Preimage

Measured images for pretext. Preimage loads and measures images before layout runs — so browser reflows don't happen, text can wrap around figures whose dimensions aren't declared, and canvas/SVG/WebGL renderers get correct fit math without relying on CSS.

First-load reflow

Real picsum photos, cache-busted on every run so the browser makes a genuine network request. Most images on the web arrive without width/height attributes; the left panel of each demo is what that looks like today, the right is what preimage makes possible.

Masonry grid

Sixty photos without declared dimensions. Naive <img> reflows as each decodes; prepare() returns measured aspects and a small inline shortest-column fill places them stably from first paint.

Editorial article with floats

Three floated figures inside flowing paragraph text. Pretext lays out the text around the figures using dimensions preimage measured up front.

Time to first sizing

One photo, three strategies side by side: naive <img>, native <img width height>, and prepare(). Timestamps show when each strategy knows the dimensions.

Outside the DOM

Places where CSS's object-fit and aspect-ratio don't apply — and preimage's numeric answers do.

Canvas fit modes

Draw an image into a canvas at the correct object-fit rect. Compares a naive stretched drawImage against layout()-computed placement.

Pretext integration

The two pretext scenarios that require numeric image dimensions pretext doesn't measure on its own.

Float a figure in a pretext column

A paragraph wrapping around a floated figure. flowColumnWithFloats drives pretext's cursor loop, narrowing maxWidth where the float is active.

Inline images in a rich-inline bubble

A chat bubble mixing text and inline icons. inlineImage() returns a pretext RichInlineItem whose extraWidth reserves the measured image's rendered width.
