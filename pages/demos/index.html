Preimage demos

Preimage is a small library of two parts: a standalone image-measurement primitive (useful wherever CSS doesn't apply — canvas, SVG, WebGL), and a pretext integration that supplies the image dimensions pretext's text-layout loop assumes as input.

The demos below are grouped the same way.

Without pretext

Strong case: drawing to canvas. The DOM has object-fit and aspect-ratio for free — you don't need a library. Canvas, SVG, and WebGL have neither.

Canvas fit

Draws an image to a canvas at the correct object-fit rect. Compares a naive drawImage(img, 0, 0, w, h) (stretched) against layout(prepared, w, h, fit) + drawImage(img, r.x, r.y, r.w, r.h) (correctly fit).

With pretext

These are the demos that actually justify the library. Pretext's README describes two use-cases — text wrapping around a figure, and rich-inline flow with atomic pills — and deliberately stops short of computing the image dimensions for either. Preimage fills that gap.

Float a figure in a pretext column

An article with a figure floated beside it. Preimage measures the image once; flowColumnWithFloats drives pretext's cursor loop, narrowing maxWidth while the float is active. Works on DOM, canvas, SVG, or WebGL targets.

Inline images in a rich-inline chat bubble

A chat bubble mixing text and small inline images. Preimage's inlineImage() returns a pretext RichInlineItem whose extraWidth reserves the measured image's rendered width, so pretext's rich-inline walker treats it as an atomic pill.

Bonus

Not part of the pretext story — a standalone justified row packer that shares the same prepared measurements.
